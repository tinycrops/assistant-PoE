Abstract

This paper documents the first direct application of our structured-reasoning research program to assistant-PoE. Rather than begin with synthetic gameplay labels, we built a benchmark from the repository’s own character traces. The task asks whether a model can identify the true immediate next state of a character from a mixed candidate set, using only the continuity structure already embedded in the project’s stat-watch logs.

We reconstructed cumulative state vectors from real logs/stat_watch/*_history.jsonl files, including 28 tracked numeric stats and 16 gear-slot occupancy signals. These states were assembled into train and evaluation ranking problems with same-character and cross-character distractors. We then compared a hand-built heuristic continuity baseline to a learned neural ranker.

The benchmark contains 20 reconstructed states across 3 characters, producing 12 training anchors and 5 held-out evaluation anchors. On this benchmark, the heuristic achieved top-1 accuracy 0.80 and MRR 0.85, while the learned model achieved top-1 accuracy 1.00 and MRR 1.00. The dataset is small and the result is preliminary, but it demonstrates a crucial point: assistant-PoE already contains enough structured temporal signal to support trainable character-intelligence experiments.

1. Context

Repository Thesis

The retrospective already argued that assistant-PoE is a character-centered intelligence system rather than a loose pile of scripts. Its strongest claim is that a character should be treated as a living memory object, not a temporary API payload.

Research Goal

We wanted to see whether that claim could be turned into a trainable benchmark using only data that already exists in the repo. In other words: can the repository’s own traces act as supervision?

4. Baselines and Model

Heuristic Baseline

The heuristic prefers:

  • same-character candidates,
  • small cumulative stat change,
  • small positive time delta.

This makes it a strong continuity prior rather than a dummy baseline.

Learned Ranker

The neural model is a small MLP over anchor-candidate pair features derived from:

  • anchor stats,
  • candidate stats,
  • signed and absolute deltas,
  • slot occupancy transitions,
  • same-character indicator,
  • time-delta features.

6. Interpretation

The most important outcome is not that a small model reached perfect held-out ranking on a tiny benchmark. The important outcome is that the repository’s own traces were structured enough to support a trainable benchmark in the first place.

What This Shows

  • Character continuity is measurable.
  • The repo already contains learnable temporal structure.
  • A learned model can beat a strong heuristic baseline.

What It Does Not Show

  • Broad PoE competence.
  • Reliable action planning.
  • General long-horizon decision quality.

Why It Matters

  • The project can now support empirical learning loops.
  • The character-centered thesis is experimentally actionable.
  • The repo has crossed from “tooling” into benchmarkable intelligence.

7. Limitations

Data Scale

This is a smoke-scale benchmark with only 20 reconstructed states and 5 held-out eval anchors. It is valuable as a traction probe, not as a definitive claim.

Eval Shape

The current held-out slice is narrow and continuity-focused. Future versions need harder negatives, more characters, and additional modalities such as market-sync and journal events.

8. Next Steps

Immediate

  • Expand the continuity benchmark with archived snapshots and future stat-watch runs.
  • Add journal and market-sync events as additional supervision.
  • Construct harder same-character distractors.

After Stabilization

  • Build a second task such as state → next action prompt.
  • Test retrieval-augmented or graph-based models over character memory.
  • Use larger Kaggle GPUs/TPUs once the benchmark contract is stronger.

9. Conclusion

This experiment is the first serious application of our structured-reasoning work to assistant-PoE, and it succeeded in the way that matters most: it found real signal in the traces the project already produces and converted that signal into a benchmark a model could learn.

That changes the status of the repository. assistant-PoE is no longer just a collection of character-aware utilities. It is now demonstrably a source of trainable character-continuity data.
